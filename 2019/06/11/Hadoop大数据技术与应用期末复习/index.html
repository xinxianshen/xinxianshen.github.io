Hadoop大数据技术与应用期末复习

发表于 2019-06-11 | | 阅读次数:

第1章 初识Hadoop大数据技术

1.大数据的4V特征：

  • 数据体量大（Volume）：大数据的数据量从 TB 级别跃升到 PB 级别。
  • 数据类型多（Variety）：大数据的数据类型包括前文提到的网络日志、视频、图片、地理位置信息等。
  • 处理速度快（Velocity）：1 秒定律。最后这一点是大数据技术与传统数据挖掘技术的本质区别。
  • 价值密度低（Value）：以视频为例，连续不间断监控过程中，可能有用的数据仅仅有一两秒。

2.Hadoop的特点：

（1）高可靠性：Hadoop按位存储和处理数据的能力强，可靠性高。

（2）高扩展性：Hadoop是在可用的计算机集群间分配数据并完成计算任务的，这些集群可以方便地扩展到数以千计的节点。

（3）高效性：Hadoop能够在节点之间动态地移动数据，并保证各个节点的动态平衡，因此处理速度非常快。

（4）高容错性：Hadoop能够自动保存数据的多个副本，并且能够自动将失败的任务重新分配。

第2章 Hadoop环境设置

1.Hadoop3种配置方式、特点与用途

（1）单机模式：单机模式中没有HDFS。

需配置的文件：

文件名称 属性名称 属性值 含义
adoop-env.sh JAVA_HOME /home/<用户名>/jdk JAVA_HOME

（2）伪分布式模式：伪分布式其实是完全分布式的一种特例，但它只有一个节点。

需配置的文件：

文件名称 属性名称 属性值 含义
adoop-env.sh JAVA_HOME /home/<用户名>/jdk JAVA_HOME
.bashrc HADOOP_HOME ~/hadoop HADOOP_HOME
core-site.xml fs.defaultFS hdfs://:8020 配置 NameNode 地址，8020 是 RPC 通信端口
↑ hadoop.tmp.dir /home/<用户名>/hadoop/tmp HDFS 数据保存在 Linux 的哪个目录，默认值是 Linux 的/tmp 目录
hdfs-site.xml dfs.replication 1 副本数
mapred-site.xml mapreduce.framework.name yarn 配置为 yarn 表示是集群模式，配置为 local 表示是本地模式
yarn-site.xml yarn.resourcemanager.hostname ResourceManager 的 IP 地址或主机名
↑ yarn.nodemanager.aux-services mapreduce_shuffle NodeManager 上运行的附属服务

（3）完全分布式模式：完全分布式模式（Fully-Distributed Mode）也叫集群模式，是真正的分布式的、由 3 个及以上的实体机或者虚拟机组成的集群。

需配置的文件：

文件名称 属性名称 属性值 含义

↑ hadoop.tmp.dir /home/<用户名>/hadoop/tmp HDFS 数据保存在 Linux 的哪个目录，默认值是 Linux 的/tmp 目录
hdfs-site.xml dfs.replication 2 副本数，默认是 3
mapred-site.xml mapreduce.framework.name yarn 配置为 yarn 表示是集群模式，配置为 local 表示是本地模式
yarn-site.xml yarn.resourcemanager.hostname ResourceManager 的 IP 地址或主机名
↑ yarn.nodemanager.aux-services mapreduce_shuffle NodeManager 上运行的附属服务
slaves DataNode 的地址 从节点 1 主机名
↑ ↑ 从节点 2 主机名

├── logs —-日志
